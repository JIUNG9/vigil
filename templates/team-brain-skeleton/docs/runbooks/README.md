# Runbooks

Index of operational procedures. One file per scenario. Each runbook follows the
template:

- **Trigger:** the alert / symptom / page that brings someone here.
- **Severity:** P0 / P1 / P2.
- **Owner:** team or person to escalate to.
- **Steps:** numbered, copy-pasteable commands.
- **Recovery:** what "good" looks like.
- **Post-mortem:** link to the doc, if there was an incident.
