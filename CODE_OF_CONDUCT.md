# Code of Conduct

This project adopts the **Contributor Covenant, version 2.1**.

The full text is at: <https://www.contributor-covenant.org/version/2/1/code_of_conduct/>

A translated version is available in many languages from the same site.

## Summary in two sentences

Everyone participating in this project — contributors, maintainers, issue commenters, anyone in a discussion thread — is expected to treat others with respect. Critique code, not people; ask before assuming intent; remember there's a human on the other end of the network.

## Scope

This Code of Conduct applies to all project spaces — the GitHub repository (issues, pull requests, discussions, commit comments, wiki), any official chat channel, project-organized events, and any time someone is acting as a representative of the project in public.

## Enforcement

If you experience or witness behavior that doesn't fit the standard above, report it. Reports go to the maintainers and will be reviewed promptly. We will not retaliate against good-faith reporters, and we will keep your report confidential.

**Report channel:** open a private security advisory at `Security → Report a vulnerability` on the repository (this is the most private channel that GitHub provides for project maintainers), or contact the maintainer listed in `pyproject.toml`'s `authors` field.

Maintainers will follow the Contributor Covenant's published enforcement ladder: warning, temporary cooldown, longer ban, permanent removal. The action taken is proportional to the behavior.

## What this means in practice

- **Disagree about code, not character.** "This API surface is confusing" is fine. "You're a bad engineer" is not.
- **Assume good faith on first encounter.** If something looks rude, it might be a non-native speaker, a bad day, or a missing comma. Ask before reacting.
- **No private harassment.** What happens in a project DM is also covered.
- **Maintainers are not exempt.** Maintainers commit to the same standard. If a maintainer's behavior falls short, report it through the channel above — other maintainers will review.
